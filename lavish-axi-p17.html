Primitive 17 — Production Config

Config & Secrets Contract

Defines how production configuration is loaded, validated, scoped, and kept separate from secrets. Secrets are referenced without being exposed in logs, traces, prompts, or serialized output.

Production Rules
⚠ Production env —> provider must have api_key_secret • store_prompt_bodies forced False • store_tool_arguments forced False
NON-EMPTY app_name, provider_name, model_name, secret_name, reference
POSITIVE timeout ≥ 1 • steps ≥ 1 • runtime ≥ 1 • total_tokens ≥ 1
NON-NEG retries ≥ 0 • cost ≥ 0 • warn_cost_usd ≥ 0
CONSISTENT total_tokens ≥ max(input, output) • warn_cost_usd ≤ max_cost_usd
REDACTED redacted_preview must differ from raw value • never serialize secrets
PRINCIPLES never in prompts • never in traces • never in durable state • never in tool results
