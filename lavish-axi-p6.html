├─ lifecycle: LifecyclePolicy

Six Policy Blocks
⚙
FilesystemPolicy
workspace_root, writable, read_only, blocked
Empty root → error
Overlap between any two sets → error
⚙
NetworkPolicy
mode (disabled/allowlist/full) + allowed_hosts
ALLOWLIST without hosts → error
DISABLED with hosts → error
⚙
ResourceLimits
CPU, memory, disk, runtime, processes
Any limit ≤ 0 → error
Default: 2 CPU, 2048 MB, 4096 MB, 600s, 20 procs
⚙
SecretPolicy
access_mode (none/explicit) + allowed_secret_names
NONE mode with secret names → error
⚙
LifecyclePolicy
mode (ephemeral/persistent) + TTL + preserve_on_failure
EPHEMERAL with TTL ≤ 0 → error
Default: 1800s TTL, preserve on failure
⚙
AuditPolicy
4 boolean flags for observability
log_tool_calls, log_resource_usage
capture_stdout_stderr, capture_fs_diff
