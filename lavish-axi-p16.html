Orchestration Flow
Step 0 ─ load contract + build context + assemble prompt └→ PROMPT_ASSEMBLY step Step 1 ─ check safety policy against tool invocation ├→ ALLOW ─→ POLICY_CHECK ✓ ├→ REQUIRE_APPROVAL ─→ APPROVAL ✓ ─→ APPROVAL_GRANTED └→ BLOCK ─→ POLICY_CHECK ✗ ─→ return blocked Step 2 ─ execute tool (mock or injected executor) ├→ success ─→ TOOL_COMPLETED └→ failure ─→ return failed Step 3 ─ run verification checks against task criteria ├→ all pass ─→ VERIFICATION_PASSED ─→ SUCCEEDED ✓ └→ any fail ─→ VERIFICATION_FAILED_TERMINAL ─→ FAILED ✗ Final ─ build RunTrace + BenchmarkRun + PipelineResult
