Validation Rules
NON-BLANK gap_id, intake_id, field_name, gap_summary, question_id, question_text, attempt_id, field_resolution_id, policy_id, session_id, decision_id, envelope_id
BLOCKING blocking severity must have blocking=True
METHOD ask_user/infer/apply_default/manual_override must include proposed_value_ref
STATE resolved/deferred/escalated/closed_unresolved need value_ref or notes
POLICY required + allow_defer must set escalation_threshold; max_attempts ≥ 1
ORDER question_order ≥ 0; active and completed question ids must not overlap
REF questions, attempts, field_resolutions, session, decision must reference valid gap_ids
DISPOSITION awaiting_response requires active questions; rejected/escalated needs reason
BLOCKERS ready_to_proceed and proceed_with_gaps cannot have unresolved blocking gaps
