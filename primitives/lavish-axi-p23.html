Enforcement Flow
ACTIVE session requires current_step_id ├→ Step completes → CheckpointRecord created (step_boundary) ├→ Before side-effect → CheckpointRecord (pre_sideeffect) + SideEffectMarker added ├→ Infra/interrupt → ResumeRequest logged └→ RecoveryDecision made: ├→ resume_from_checkpoint → load snapshot, continue from target step ├→ restart_step → rewind to target step, potentially revalidate ├→ require_revalidation → replay step with human review gate └→ abort_resume → session failed, no replay
Validation Rules
NON-EMPTY session_id, run_id, checkpoint_id, request_id, decision_id, step_id, marker_id, snapshot_id, storage_uri
ACTIVE ACTIVE session must have current_step_id
COMMITTED side_effect_committed=True requires idempotency_key
REQUIRES KEY REQUIRES_IDEMPOTENCY_KEY step mandates idempotency_key on all markers
ABORT ABORT_RESUME must not target a step
RESUME RESUME_FROM_CHECKPOINT / RESTART_STEP must target a step
RATIONALE RecoveryDecision rationale must be non-empty
