Status Lifecycle
PENDING Task created, not yet ready for execution | READY All inputs available, acceptance criteria defined, waiting for dispatch | RUNNING Execution in progress | +--+--+--+--+--+ | BLOCKED Dependency or resource unavailable | NEEDS_INPUT Missing required input data | NEEDS_APPROVAL Pending human reviewer decision | FAILED Execution terminated with error | CANCELLED Abandoned before completion +--+--+--+--+--+ | COMPLETED Acceptance criteria satisfied, outputs delivered
Validation Rules
NON-EMPTY task_id, title, objective, envelope_id, run_id, input/output/constraint/criterion IDs, source_ref, category, description
NO PLACEHOLDER title ('untitled', 'tbd', 'todo', 'task', 'new task') and objective ('tbd', 'todo', 'objective', 'goal') rejected
NO SCOPE OVERLAP in_scope and out_of_scope must not contain the same item
EXECUTABLE NEEDS CRITERIA READY, RUNNING, COMPLETED statuses require at least one acceptance criterion
WHITESPACE STRIPPED All string fields trimmed; blank items in scope lists rejected
Failure Mode Semantics
STOP Halt execution and report failure back to caller
RETRY Retry the task (up to configured limit in implementation)
ESCALATE Escalate to a human or higher authority
REQUEST_INPUT Move to NEEDS_INPUT status and request missing data
