Three-Model Architecture
■
ToolDefinition
Static tool metadata
ToolDefinition
├─ name: str
├─ description: str
├─ input_schema: Dict
├─ output_schema: Optional[Dict]
├─ requires_approval: bool
├─ execution_mode: ExecutionMode
├─ tags: List[str]
└─ enabled: bool
■
ToolInvocation
One model-requested call
ToolInvocation
├─ invocation_id: str
├─ tool_name: str
├─ arguments: Dict
├─ requested_by_task_id: str
├─ requested_by_agent: str
└─ approval_state: ApprovalState
■
ToolResult
Execution outcome
ToolResult
├─ invocation_id: str
├─ tool_name: str
├─ status: ToolStatus
├─ output: Optional[Dict]
├─ error: Optional[str]
├─ artifacts: List[str]
├─ started_at: Optional[str]
└─ finished_at: Optional[str]

Validation Rules
⚙
ToolDefinition
name, description must not be empty
input_schema must not be empty
⚙
ToolInvocation
invocation_id, tool_name, task_id, agent must not be empty
Cannot start as already APPROVED → must go through PENDING
⚙
ToolResult — Status Consistency
SUCCESS must have output or artifacts (can't be empty success)
ERROR must include an error message
BLOCKED must include an error message
