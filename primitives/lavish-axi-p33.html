Context Trust Model
TRUSTED Authored by the harness itself (system instructions, verified policy notes) INTERNAL_UNVERIFIED Sourced from internal systems but not independently verified (tool output, retrieval) EXTERNAL_UNTRUSTED Sourced from outside the system boundary (web content, user uploads)
Validation Rules
NON-EMPTY call_id, run_id, agent_id, model_ref_id, model_name, response_ref, block_id, content_ref, provenance_id, source_type, source_ref
CONFIDENCE confidence must be between 0.0 and 1.0 if present
TEMPERATURE temperature must be between 0.0 and 2.0 if present
TOP_P top_p must be between 0.0 and 1.0 if present
TOKENS max_output_tokens must be >= 1; prompt_tokens, output_tokens, latency_ms must be >= 0
ORDER prompt_blocks order is preserved (list semantics)
Prompt Assembly Principles
STRUCTURED Context is assembled as explicit ordered blocks, not a single opaque blob
SEPARATED Instructions (system_instruction) and evidence (retrieval, tool_output) are distinct block types
PROVENANCE Every block can carry source_type, source_ref, timestamp, and confidence
UNTRUSTED Externally sourced context is marked external_untrusted by default (injection resistance)
