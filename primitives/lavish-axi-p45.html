Validation Rules
NON-BLANK step_id, loop_id, envelope_id, intent_id, objective, selection step_id/loop_id, verification_id, action_id
ACCEPTANCE acceptance_criteria must be non-empty for executable work
TERMINAL COMPLETED/FAILED/CANCELLED requires ended_at
SUCCESS completed+success requires verification.passed=True
FAILURE failure/no_progress outcomes require stop_reason
NO PROGRESS no_progress requires non-empty next_step_hint
STATE DELTA state-changing steps must record artifact updates
DELEGATION delegation_request actions require delegation_allowed=True
CONFIDENCE passed verification requires confidence ≥ 0.5
