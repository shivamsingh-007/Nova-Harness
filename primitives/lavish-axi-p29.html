Evaluation Flow
TRIGGER Action reaches guardrail boundary (prompt / tool / memory / output / step) POLICY LOOKUP PolicyRef matched for target type and context EVIDENCE COLLECTION Signals gathered from classifiers, regex, analyzers, scope checks FINDINGS Structured findings with severity and linked evidence RISK ASSESSMENT Overall risk level from findings DECISION allow | deny | modify | review | quarantine | defer OBLIGATIONS Attach approvals, masking, scope reductions, etc. ENVELOPE Signed decision with execution_allowed flag
Guardrails evaluate BEFORE execution — never after. Guardrails constrain and enforce. Evaluations measure quality. Different contracts.
Validation Rules
NON-EMPTY evaluation_id, decision_id, target_ref, policy_id, policy_name, rationale, decided_by, signal_id, signal_type, source_ref, summary, finding_id, category
CONFIDENCE must be between 0 and 1 inclusive if present
RESTRICTIVE DENY and QUARANTINE must set execution_allowed=False
SEVERITY HIGH and CRITICAL findings must include at least one evidence signal
POLICY PolicyEvaluation must always include a PolicyRef
FINDINGS Must be structured objects, not raw text
Decision Semantics
ALLOW execution_allowed=True   Normal execution with optional log_only
DENY execution_allowed=False   Blocked entirely, no execution
MODIFY execution_allowed=True   Content transformed (masking, etc.)
REVIEW execution_allowed=True   Proceeds pending human approval
QUARANTINE execution_allowed=False   Held for investigation, no execution
DEFER execution_allowed=False   Decision postponed
