Receipt Anatomy
IDENTITY receipt_id, request_id, run_id, agent_id, tool_id, action LINKAGE task_id?, trace_id?, policy_decision_ref?, approval_ref? TIMING requested_at, started_at?, completed_at?, duration_ms? RESULT status, outcome_summary, result_references[], side_effects[] ERROR error_code?, error_summary? (required when FAILED/TIMED_OUT/BLOCKED) INTEGRITY mode, canonical_hash?, signature_ref?, previous_receipt_hash?
Integrity Mode Requirements
NONE No integrity checks (default)
HASH_ONLY Requires canonical_hash
SIGNED Requires signature_ref
CHAINED Requires previous_receipt_hash
Validation Rules
NON-EMPTY receipt_id, request_id, run_id, agent_id, tool_id, action, requested_at, result_ref_id, result_type, outcome_summary, envelope_id, summary, target_refs entries
DURATION duration_ms must be non-negative if present
TEMPORAL completed_at must not be earlier than started_at
FAILURE FAILED, TIMED_OUT, BLOCKED statuses require non-empty error_summary
INTEGRITY SIGNED requires signature_ref, CHAINED requires previous_receipt_hash, HASH_ONLY requires canonical_hash
