Synthesis Pipeline
SELECT SOURCES → NORMALIZE → RESOLVE CONFLICTS → ASSEMBLE RESULT → FINALIZE
accepted / rejected    schema_map/dedup    rank/resolve/review    merge & validate    approved / rework/reject
Validation Rules
NON-BLANK envelope_id, source_id, source_ref, eligibility_status, normalization_id, input_ref, normalized_ref, conflict_id, conflict_summary, decision_id, validation_summary_id, finalization_id, output_id, result_summary
ELIGIBILITY at least one source must be eligible (accepted)
REJECTED rejected-only inputs cannot become approved or released outputs
RELEASED released status requires release_ready=True
ACCEPT accept dispositions require at least one final_output_ref
REWORK/REJECT must include approval_reason
CONFLICTS unresolved conflicts must set review_required on validation summary
QUALITY quality_score bounded 0.0 ≤ v ≤ 1.0
CONFIDENCE confidence bounded 0.0 ≤ v ≤ 1.0
