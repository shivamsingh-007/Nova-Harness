Grader Strategy
1. DETERMINISTIC primary grader — output matching, exit code, file comparison
2. HUMAN REVIEW for calibration and ambiguous tasks
3. LLM JUDGE only after calibration — known error rate needs validation
Validation Rules
Non-empty ids suite, task, run, grader, reference
At least one task per suite
At least one grader + metric per run
pass_rate between 0 and 1
Non-negative latency / cost / retries / safety violations
finished_at required for PASS or FAIL verdicts
Non-empty pass_condition on every grader
Default Metrics
task_success did the agent complete the task
verification_pass did the verification gate pass
retry_count how many retries were needed
latency_seconds p95 / average end-to-end time
cost_usd model inference cost per task
safety_violations how many safety policy blocks
tool_success did all tool calls succeed
