Context governance
Scope/status/source/trust labeling, confidence/score ranges, positive rank, retention durations, uniqueness across snapshots and envelopes
Context Lifecycle
RETRIEVAL query → ranked hits → memory items loaded
ASSEMBLY snapshot + memory items → context blocks by trust/purpose
RETENTION policy decides keep_until, expire_after_days, promote_to_project, summarize_before_store
STATUS active → stale → archived → deleted (with audit trail)
Trust Levels in Context
TRUSTED Verified system/tool output or manually curated project memory
INTERNAL_UNVERIFIED Stale items, deprecated references, unconfirmed internal data
EXTERNAL_UNTRUSTED User-provided content, web data, unverified external sources
