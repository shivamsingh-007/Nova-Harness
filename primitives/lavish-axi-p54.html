Parallel Workflow
PARALLEL REQUEST
└ Branch 1 (completed) → JOIN → MERGE (success)
└ Branch 2 (completed) → JOIN → MERGE (success)
└ Branch 3 (failed) → JOIN → MERGE (partial)
└ Branch 4 (blocked) → JOIN → MERGE (needs_review)
Validation Rules
NON-BLANK envelope_id, parallel_request_id, objective, branch_id, scope_summary, branch_output_id, conflict_id, conflict_summary, join_requirement_id, join_id, merge_id
BRANCHES at least one branch, branch IDs unique per request
COMPLETION required_completion_count cannot exceed total required branches
FIRST_SUCCESS wait_for_first_success must not require all branches
MERGE merge results require completed or partial join state
CONFLICT conflict and output records must reference valid branch IDs
CONFIDENCE confidence bounded 0.0 ≤ v ≤ 1.0
