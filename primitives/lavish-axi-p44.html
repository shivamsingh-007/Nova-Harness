Validation Rules
NON-BLANK envelope_id, objective, current_objective, loop_id
POSITIVE iteration ≥ 1, max_iterations > 0
NON-NEGATIVE max_no_progress_iterations ≥ 0
TERMINAL STATUS COMPLETED/EXHAUSTED/FAILED requires stop_reason
STOP REASON non-null stop_reason requires terminal status
ACTIVE LOOP RUNNING status requires non-empty memory.next_action
VERIFICATION COMPLETED/EXHAUSTED with verification_required needs verification_result on all cycles
Iteration Sequence
INTAKE → PLAN → EXECUTE → VERIFY → UPDATE_STATE → STOP_CHECK → (continue or) HANDOFF
