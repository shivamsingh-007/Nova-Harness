RunState — 7 Records + 4 Enums
■
RunState
run_id + task_id + status + step list + checkpoints + resume pointer + timestamps
RunState
├─ run_id: str (non-empty)
├─ task_id: str (non-empty)
├─ status: RunStatus
├─ steps: List[StepRecord]
├─ checkpoints: List[CheckpointRecord]
├─ resume: ResumePointer
├─ created_at: str (non-empty)
└─ updated_at: str (non-empty)
Seven Records
■
CheckpointRecord
checkpoint_id, run_id, step_id, summary, created_at, state_version
Empty checkpoint_id → error
Empty run_id → error
Empty created_at → error
Default state_version = 1
■
StepRecord
step_id, name, status, tool_calls, artifacts, retry, timestamps
Empty step_id or name → error
SUCCESS/FAILED without finished_at → error
retry_count > max_retries → error
■
ToolCallRecord
tool_call_id, tool_name, status, arguments, output/error, timestamps
Empty tool_call_id or tool_name → error
ERROR without error message → error
■
ArtifactRecord
artifact_id, artifact_type, path, description, created_at
Empty artifact_id → error
Empty path → error
■
RetryState
retry_count, max_retries, last_error, last_attempt_at
Defaults: count=0, max_retries=0
Cross-validated in StepRecord
■
ResumePointer
next_step_id, last_checkpoint_id, can_resume, resume_reason
can_resume=False without reason → error
Default: can_resume=True

Validation Rules
Non-empty IDs run_id, task_id, step_id, tool_call_id, artifact_id, checkpoint_id
Non-empty names step name, tool name
Terminal check SUCCESS/FAILED step requires finished_at
Retry bound retry_count must not exceed max_retries
Error presence ERROR tool call requires error message
Resume reason can_resume=False requires resume_reason
Timestamps created_at, updated_at on RunState; created_at on CheckpointRecord
Checkpoint Moments (v1 Guidance)
▶ After each completed model turn
▶ After each completed tool call
▶ After each completed step
▶ Before pausing for approval
▶ Before marking failure terminal
