Validation Rules
NON-BLANK envelope_id, bundle_id, manifest_id, profile_id, name, key, setting_id, value_type, secret_id, secret.name
DUPLICATE duplicate config keys rejected within a profile
DUPLICATE duplicate secret names rejected within a profile and manifest
REQUIRED required config keys must have an entry in bundle.values
MAPPING runtime settings must reference declared config keys
SENSITIVE sensitive settings must not carry a default_value
SECRET OVERRIDE defaults must not silently override required secrets
MATCH secrets_manifest.profile_id must match bundle.profile.profile_id
Environment Profile Lifecycle
LOCAL → relaxed defaults, env-var secrets, localhost
DEV → shared dev infra, env-var or file secrets
TEST → CI-friendly in-memory defaults, test-specific secrets
STAGING → mirror production with vault secrets, info logging
PRODUCTION → strict required keys, secret-manager/KMS, warn logging
       ↓
STARTUP VALIDATION → all required keys present, no duplicates, no secret leaks
