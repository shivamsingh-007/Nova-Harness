Delegation Chain Flow
AgentIdentity defines who the agent is and who owns it
OwnerRef (team-eng, user-shiva) owns AgentIdentity
└→ DelegationContext preserves on_behalf_of chain
├→ delegated_by_principal_id = user-shiva
├→ on_behalf_of_user_id = user-shiva
├→ authority_scopes = [read_file, write_file]
└→ delegation_reason = "Manual CLI request"

→ RunOwnership connects run to agent, initiator, and owning principal
→ AuthorityAssertion records effective scopes for the run
├→ bounded = scoped to specific tools/paths
├→ delegated = acting on behalf of a user
└→ autonomous = self-authorized within defined scope
Validation Rules
NON-EMPTY agent_id, agent_name, owner_id, owner_type, scope_id, scope_type, scope_value, run_id, requested_by_id, owning_principal_id, assertion_id
OWNERSHIP Every AgentIdentity must have an OwnerRef
LIFECYCLE ACTIVE/SUSPENDED identities must have review_due_at
DELEGATION on_behalf_of requires delegation_reason • delegated requires at least one authority_scope
RUN USER-initiated runs must have delegation_context
SCOPES AuthorityAssertion must have at least one effective_scope
