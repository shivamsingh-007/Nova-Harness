Interface
⚙
StateDriver
Runtime wrapper over OrchestrationStateMachine
.machine OrchestrationStateMachine (from P10)
.current_state_id RunState default: CREATED
.history List[TransitionRecord] every attempt logged
.is_terminal bool terminal? no more events
.available_events() List[RunEvent] valid events from current state
.trigger(event, guards) RunState transition or raise
📖
TransitionRecord
Each trigger attempt logged in history
from_state RunState state before event
event RunEvent triggered event
to_state RunState state after (or same if guard blocked)
timestamp str (ISO) when trigger was called
guard_results Dict[str, bool] guard name → result
decision str "allowed" or "blocked by guard: X"
Driver Behavior
trigger(event, guard_results?)
  1. Terminal check → raise if terminal
  2. Find matching transition → raise if no match (shows available)
  3. Evaluate guards → reject if any guard fails (stays put, logged)
  4. Record + transition → append to history, update state
Full Traversal Example
CREATED ─ START_RUN ─→ ASSEMBLING_PROMPT ─ PROMPT_READY ─→ MODEL_PENDING ─ MODEL_RESPONSE_RECEIVED ─→ MODEL_COMPLETED ─ NO_TOOL_NEEDED ─→ VERIFYING ─ VERIFICATION_PASSED ─→ SUCCEEDED (terminal) Alternative: TOOL_CALL_REQUESTED → APPROVAL_REQUIRED → APPROVAL_REJECTED → FAILED Alternative: TOOL_FAILED_RETRYABLE → RETRY_PENDING → RETRY_ALLOWED{retry_budget:ok} → ASSEMBLING_PROMPT
