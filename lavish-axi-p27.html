Retention Boundaries
SESSION MEMORY → EPHEMERAL or SHORT_LIVED only WORKING MEMORY → EPHEMERAL or SHORT_LIVED only LONG_TERM MEMORY → STANDARD, EXTENDED, or LEGAL_HOLD only
All non-ACTIVE lifecycle states (INVALIDATED, SUPPRESSED, TOMBSTONED, EXPIRED, QUARANTINED) → Default non-retrievable in implementation Scope boundary before ranking: ├→ Filter by ExecutionContext (run/session/user/project/workspace/tenant) ├→ Filter by lifecycle_status (non-ACTIVE excluded by default) └→ Filter by retention policy (EXPIRED records excluded)
Validation Rules
NON-EMPTY memory_id, scope_id, content_ref, decision_id, reason
RETENTION SESSION/WORKING must not use EXTENDED or LEGAL_HOLD
RETENTION LONG_TERM must not use EPHEMERAL or SHORT_LIVED
TTL ttl_seconds must be positive if provided
SCOPE MemoryScope always required on MemoryRecord
PURPOSE Purpose must be explicit (enum, not freeform)
