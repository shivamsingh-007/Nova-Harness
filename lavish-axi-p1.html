Three-Layer Instruction Architecture

Separate permanent behavior from project rules and task-time guidance. Prevents rule collisions, reduces noise, and makes debugging easier when the agent misbehaves.

Global Layer — ~/.config/opencode/AGENTS.md
# Global AGENTS.md
## Working Style
- Be concise, clear, and structured.
- Solve one bounded problem at a time.
## Coding Behavior
- Minimize scope of edits.
- Prefer existing project patterns.
## Verification
- Run the smallest relevant check first.
## Safety
- Avoid destructive actions without confirmation.
Project Layer — ./AGENTS.md
# AGENTS.md
## Project Identity
- Harness system for AI coding agents
## Repository Rules
- Build one primitive at a time
- Keep modules independent and composable
## Safety Boundaries
- No secrets, CI/CD, or infra changes
## Verification Standard
- Task not done until receipts exist
## Expected Response Shape
- 6-part structure for every task
Risks & Mitigations
⚠
Rule Duplication
Same rule in global + project
Creates confusion about which evolves
Mitigation: project rules override global; keep global generic
⚠
Layer Leakage
Project rules in global file
Affects unrelated repos
Mitigation: strict placement test before adding rules
⚠
Task Details in Permanent Files
Makes instructions noisy and stale
Long-lived files accumulate transient rules
Mitigation: task details stay in runtime prompt only
⚠
Conflicting Rules
One layer says minimal, other says full refactor
Reduces consistency
Mitigation: precedence: task > project > global
